Why this exists

I wanted a faster way to isolate filesystem issues and recover important data from a host machine without touching the host OS.

This started with my Dad: his Windows machine had a completely full C:\ drive, which made it basically unusable. He also didn’t want to lose what he had archived.

So I built a custom live image equipped with the filesystem tools we use for diagnosis and recovery. It’s designed to let you mount the host disks, point your CLI tools at the host machine’s files, back up what matters, and then (once you’re safe) do things like partition resizing so it’s not a recurring problem.

Safety / YMMV

I’ve done all of this manually and it’s very easy to make mistakes with disk operations.

There is no assurance of data safety. Make sure you understand what you’re asking the tools to do. If you’re not sure, stop and make a backup first.

Over time, I may add more tools to validate that the host hardware itself looks healthy and safe to keep using. For now, this is what’s included as of today.

Features

Development Tools

Cursor IDE and Antigravity IDE pre-installed and ready to use

Filesystem Support

NTFS, exFAT, BTRFS, XFS, HFS+ and more with full tool support

Network Ready

WiFi control via NetworkManager, Firefox browser included

Disk Tools

Disk usage visualizers (baobab, ncdu, duf) for storage analysis

Fast Boot

Direct boot with custom splash - no menu, straight to desktop

Quick Start Instructions

  1. Download the ISO file using the button above
  2. Flash to a USB drive (8GB+ recommended):
    sudo dd if=nixos-custom.iso of=/dev/diskX bs=64M status=progress
  3. Boot from USB (usually F12, F2, or Del during startup)
  4. System boots directly into GNOME - no login required!
  5. Default password is nixos if needed (sudo is passwordless)
